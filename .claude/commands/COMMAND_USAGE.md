# Command Usage Reference

Quick reference for using workflow commands with proper syntax and organization.

## Command Syntax

### `/research-code [description]`

Research codebase architecture, patterns, and integration points. **Generates issue name** from description.

**Examples:**
```bash
/research-code Research authentication patterns for OAuth implementation
/research-code Investigate memory management in data processing
```

**Generates:** Issue name (e.g., `add-oauth-auth`, `fix-memory-leak`)
**Creates:** `.claude/planning/{issue-name}/CODE_RESEARCH.md`, `STATUS.md`

---

### `/issue-planner {issue-name}`

Create implementation plan and technical specification. **Uses issue name** generated by research-code.

**Examples:**
```bash
/issue-planner add-oauth-auth
/issue-planner fix-memory-leak
```

**Creates:** `IMPLEMENTATION_PLAN.md`, `PROJECT_SPEC.md` | **Updates:** `STATUS.md`

---

### `/execute-plan {issue-name}`

Systematically implement following the documented plan. **Uses issue name** from previous commands.

**Examples:**
```bash
/execute-plan add-oauth-auth
/execute-plan fix-memory-leak
```

**Creates:** Implementation code, tests | **Updates:** `STATUS.md`

---

### `/review-code {issue-name}`

Run automated checks and manual code review. **Uses issue name** from previous commands.

**Examples:**
```bash
/review-code add-oauth-auth
/review-code fix-memory-leak
```

**Creates:** `CODE_REVIEW.md` | **Updates:** `STATUS.md`

---

## Issue Name Format

**Auto-generated by `/research-code`** from your feature description.

**Rules:**
- kebab-case (lowercase-with-hyphens)
- Concise (2-5 words)
- Descriptive

**Examples:**
- ✅ `add-oauth-auth`, `fix-memory-leak`, `refactor-api-layer`
- ❌ `AddOAuthAuth` (not kebab-case), `fix` (too vague), `add-new-auth-system-with-oauth2` (too long)

## File Organization

```
.claude/planning/
├── add-oauth-auth/          # Issue-based directory
│   ├── STATUS.md           # Central progress tracker
│   ├── CODE_RESEARCH.md    # Research findings
│   ├── IMPLEMENTATION_PLAN.md
│   ├── PROJECT_SPEC.md
│   └── CODE_REVIEW.md
└── fix-memory-leak/
    ├── STATUS.md
    └── CODE_RESEARCH.md
```

**STATUS.md** = Single source of truth for workflow progress, updated by all commands.

---

## Complete Workflow

```bash
# 1. Research (generates issue name: add-oauth-auth)
/research-code Research OAuth2 authentication patterns with Google and GitHub

# 2. Plan (uses generated issue name)
/issue-planner add-oauth-auth

# 3. Implement
/execute-plan add-oauth-auth

# 4. Review
/review-code add-oauth-auth
```

**Check progress anytime:** `cat .claude/planning/add-oauth-auth/STATUS.md`

---

## Benefits

- **Multi-feature support** - Work on multiple issues simultaneously
- **Organized artifacts** - All docs for a feature in one directory
- **Progress tracking** - STATUS.md shows complete workflow state
- **Context continuity** - Commands read previous phase outputs
- **Team collaboration** - Share STATUS.md for clear progress visibility
